# Ferric Canvas v0.3 Self Review

## Task summary

Updated the vendored Ferric Canvas package to upstream revision `a3e740375ae390b06292fa2a481de7fbafa1c610`, adopted its incremental transaction facade, and completed Coart-owned interaction, media, page, Slides, HTML, layer, and navigation workflows with contextual controls.

## Corrections and user feedback

- The user emphasized that image crop/alt/annotation and Slides export/play belong in context toolbars. The implementation therefore added media actions and Slides layout/export/play to selection-dependent controls rather than expanding the fixed global toolbar.
- Upstream exposure was audited before implementation. Only mutation primitives were adopted from Ferric; clipboard, pages, media editing, Slides, HTML DOM editing, layers, and minimap remain Coart product-layer code.

## Failure -> cause -> fix

- The first vendor copy omitted wildcard-matched generated files because `Copy-Item -LiteralPath` does not expand wildcards. Files were recopied by enumerating each generated artifact, then source/vendor WASM hashes were compared successfully.
- The first full quality run failed the Widget intrinsic-height guard. The exported Slides HTML template contained a separate `html,body{height:100%}` string, which triggered the host Widget guard even though the real Widget CSS retained `min-height:640px`. The deck template now uses separate `:root` and `body` rules, preserving both contracts.
- Initial TypeScript validation rejected a union-typed Slides name. A runtime string guard now derives the deck name before HTML export.
